Digital Monoprinting

I make web-based tools that generate images. Currently**, I am working on a tool that:

  1. Allows me to record a doodle.
  2. Creates an image that uses the drawing as a background layer.
  3. Plucks random sections of the drawing to be redrawn
  4. A random selection of color functions are used to work over areas of the drawing

The tool doodles over a doodle of a doodle.

Randomness

The extent of the randomness used can be refined with settings that give me more control. I can set things up so new images are constantly generated, I see them arriving and delete them if they donâ€™t seem to work.

Digital Monoprinting: In practice, what I do is reminds be a lot of making monoprints because it's not clear quite what the image will look like until peeling back the paper.

I struggle to understand why some of my generated images appear to be more successful, but I like working in this kind of space. Sometimes it's like someone else made the image.

Collaboration: I'd like other people to be able to create their own gallery on my website using my tools. It could get complicated, but the main thing would be that they would provide a drawing on screen then pick settings and make sections as images are generated. I have tried to create programms that generate images without a drawing as a starting place*, but at them moment I am more interested in feeding a drawing in at the start

I find I tinker with the code rather than use it to create. I think it's time to stop coding and focus on using the tools I have built to create substantial sets of images. I think it's reached that point of maturity. I am also interested in getting back to making physical artwork, probably with the plotter, maybe with paint.
